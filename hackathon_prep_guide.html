Round 2 Strategy & Defense Guide

1. The Core Engineering Pitch

To win Round 2, you need to convince the judges that this project is a highly engineered, resilient production-grade architecture designed to solve real corporate problems.

The Architecture Stack Breakdown:

  • FastAPI (Python): High-performance asynchronous framework perfect for streaming real-time events and telemetry using Server-Sent Events (SSE).
  • React + Vite: Lightning fast developer build tooling providing modular UI components and responsive layouts.
  • SQLite + SQLAlchemy: Isolated local SQL storage preserving complete session data across tasks, with ORM safety loops.

2. AI Utilization & Telemetry Integration

Explain how AI models are actively integrated and governed in the project:

  • The Orchestrator Router: The orchestrator uses gemini-1.5-flash with specific system instructions to dynamically parse requests and delegate them to specialist agents.
  • Token Telemetry & Virtual Costing: Telemetry calls inspect model response objects for usage_metadata and calculate virtual costs based on input ($0.075/1M) and output ($0.30/1M) token rates.
  • AI Diagnostic Chatbot: Reads logs from the database, builds context payloads, and generates diagnostic advice for failure events.

3. Production-Grade Resiliency Features

1. Loop-Breaker Anomaly Interceptor

AI agents can get stuck in infinite retry loops due to resource timeouts, burning token budgets. The observer tracks repeat counts of agent logs. If an agent attempts the same operation 5 times consecutively, the breaker automatically terminates execution, marks the task failed, and triggers a dashboard alert banner.

2. Human-in-the-Loop Governance Center

High-risk operations (e.g. allocating budgets > $1,000) are paused in a pending approval state. The task halts execution and registers a record in the approvals table, resuming only when an operator clicks Approve.

3. Step-by-Step Debugger Replay

Every log line is stored in SQLite. Developers can click Replay Trace on any past task to step through logs frame-by-frame, creating a full timeline replay for post-incident audits.

4. Anticipated Judges Q&A

Q1: "Why did you build a simulator instead of just running CrewAI?"
Answer: "We built a dual-track engine. If a Gemini API Key is present, it runs live reasoning calls. If not, it falls back to a deterministic state simulator. We engineered this simulator because live agent loops are highly volatile, costly, and can fail due to internet latency or rate limits during a high-stakes demo. The simulator guarantees we can demonstrate the exact p99 failure states—such as loop anomalies and human approvals—predictably, while maintaining database integrity."
Q2: "How is thread safety handled when updating database records from background agent tasks?"
Answer: "FastAPI handles requests in a thread pool. To prevent thread collisions in SQLite, we use SQLAlchemy's SessionLocal to generate scoped database sessions for each background task runner using try...finally resource cleanups. This keeps connection sessions isolated, preventing database locks during concurrent runs."
Q3: "How does the Lineage Graph update dynamically?"
Answer: "The lineage graph is not hardcoded. The frontend queries the /api/lineage endpoint, which reads the agents and agent_dependencies tables in SQLite. When you register a new agent, the table is updated, and the React Flow component automatically reconstructs the node layout and links in real-time."
